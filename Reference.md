\f0\fs24 \cf0 # FRAMEWORK: THE OPERATIONAL WELLNESS MATRIX\
\
## OVERVIEW\
When evaluating any AI tool, model, or agentic workflow for a Small-to-Medium Business (SMB), you must run the tool through this evaluation matrix. This is your primary "Reference Concept" for determining if a tool is a Go or No-Go.\
\
---\
\
## THE 4 QUADRANTS OF SMB AI EVALUATION\
\
### 1. Setup Friction (The Implementation Tax)\
*   **The Check:** Can a non-technical business owner set this up in one afternoon using standard tutorials, or does it require a developer, command-line interface, and API keys?\
*   **The Rule:** High friction tools require extreme justification. If it requires custom code, flag it immediately as a high barrier to entry.\
\
### 2. Maintenance Debt (The Breakage Risk)\
*   **The Check:** When the underlying AI model updates or the API changes, will this workflow silently break? Who has to fix it?\
*   **The Rule:** SMBs rarely have dedicated IT staff. You must heavily favor resilient, visually-mapped, low-code platforms over fragile custom scripts.\
\
### 3. Tangible ROI (The Time-to-Value Ratio)\
*   **The Check:** Does the time saved by the AI significantly outweigh the time spent reviewing the AI's mistakes?\
*   **The Rule:** If an AI tool takes 15 minutes to configure and prompt just to save 10 minutes of manual typing, it has a negative Operational ROI. The tool must provide asymmetric returns.\
\
### 4. Data Safety (The Trust Factor)\
*   **The Check:** Does the tool's enterprise or paid tier explicitly state it does NOT train its models on user data? \
*   **The Rule:** You must inform the user with full context before recommending a tool that forces a business owner to put sensitive client data (like financial records, legal documents, or private emails) into a public training pool.}
